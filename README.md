# UniHelp – Flask NLP Backend

REST API backend for the RSU Smart Student HelpDesk Chatbot.
Built with Python 3.12 + Flask. Deployed on Railway.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Web framework | Flask 3.0 |
| Production server | Gunicorn |
| NLP pipeline | Custom (tokenizer, classifier, entity extractor, response engine) |
| Knowledge base | Curated RSU data + optional web-scraped content |
| Web scraping | requests + BeautifulSoup4 |
| Deployment | Railway |

---

## Project Structure

```
SERVER/
├── app.py                  # Flask entry point, API routes
├── Procfile                # Railway/Heroku start command
├── requirements.txt        # Python dependencies
├── runtime.txt             # Python version for Railway
├── scraper.py              # One-time RSU website scraper
│
├── nlp/
│   ├── tokenizer.py        # Lowercasing, stop-word removal, tokenization
│   ├── intents.py          # 14 intent definitions with keywords and patterns
│   ├── classifier.py       # Keyword scoring + prefix stemming classifier
│   ├── entity_extractor.py # Extracts department, level, semester, scholarship
│   └── response_engine.py  # Generates formatted responses from knowledge base
│
└── data/
    ├── knowledge_base.py   # Curated RSU data (fees, faculties, contacts, etc.)
    └── scraped_rsu.json    # Auto-generated by scraper.py (optional)
```

## Local Setup

```bash
cd SERVER

# Install dependencies (Python 3.12 required)
py -3.12 -m pip install -r requirements.txt

# Run the development server
py -3.12 app.py
```

The API will be available at `http://localhost:5000`.

## API Endpoints

### GET /api/health
Health check.

**Response:**
```json
{ "status": "ok", "service": "UniHelp API" }
```

### POST /api/chat
Send a student message and receive an NLP-generated response.

**Request body:**
```json
{ "message": "What are the admission requirements?" }
```

**Response:**
```json
{
  "intent": "admission",
  "confidence": 1.0,
  "entities": { "department": "engineering" },
  "response": "**Admission – Rivers State University**\n\n...",
  "suggestions": ["What is the JAMB cutoff mark?", "..."]
}
```

## Evaluation

A test suite of **87 questions** across all 14 intents is included. It measures
intent-classification accuracy and average response time.

```bash
# 1. Start the Flask server first
py -3.12 app.py

# 2. In a second terminal, run the evaluator
py -3.12 evaluate.py
```

To evaluate the deployed Railway server instead:

```bash
py -3.12 evaluate.py --url https://web-production-608a0.up.railway.app
```

**Output includes:**
- Pass / fail for every test case with predicted intent and confidence
- Per-intent accuracy table with average response time
- Overall accuracy percentage and efficiency rating
- Full results saved to `evaluation_report.json`

## Running the Scraper

The scraper fetches content from the RSU website and saves it to
`data/scraped_rsu.json`. Run it once to update the knowledge base with
live website content.

```bash
py -3.12 scraper.py
```

The knowledge base loads the scraped JSON automatically when `app.py` starts.

---

## Deployment (Railway)

1. Push the `SERVER/` folder to a GitHub repository.
2. Create a new Railway project and connect the repository.
3. Railway detects `Procfile` and runs: `gunicorn app:app`.
4. Set the `FLASK_ENV=production` environment variable in Railway settings.
5. Copy the Railway URL and update `API_URL` in the frontend's `chatService.js`.
